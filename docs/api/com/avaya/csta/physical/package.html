Contains constants used by Communication Manager to represent button types/functions, button addresses/IDs, lamp modes, and ringer types/patterns used in PhysicalDeviceServices. The application must use these values when specifying buttons, lamps and ringers in service requests, such as when specifying which button to press or which lamp to obtain status on. The application may also have to interpret values sent to it in responses and events. For example, if you requested button information for all the buttons on a device using the getButtonInformation() request, then the response would contain a list of all the buttons on that device, each one identified by its button ID and button function.

Note the following mapping between CSTA field names and Communication Manager terms:
CSTA Field Name Equivalent Communication Manager Term Description
buttonID or lampID button address Integer value. Both a buttonID and a button address represent the location of the button on the phone. If the button has a lamp, the lampID is the same value as the buttonID.
buttonFunction button type String value. Both buttonFunction and button type represent the function or feature assigned to the button, such as "CALL_FWD" for the call forward feature and "SEND_CALLS" for the send all calls feature.
lampMode lamp mode Integer value. A value representing the mode of the lamp, such as 0 for broken flutter, 1 for flutter, 2 for off, 3 for steady, 4 for wink, 5 for unknown, etc.
ringPattern ring type Integer value. The value of the ringing pattern. The meaning of ringing patterns and the number of supported patterns is device specific.
