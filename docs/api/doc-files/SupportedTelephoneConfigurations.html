Telephone Configurations Controlled by Communication Manager API

The Communication Manager API can control the following types of Communication Manager telephones and extensions when they are administered for softphone access:
Telephone Configuration Administered Set Type Administered Port Address Affect on Physical Telephone
DCP telephone Any DCP type that can be administered for softphone access Physical port address, such as 1B0201. If an application requests exclusive control, the telephone either goes dead or becomes a TTI set, depending on how it is administered. After the application relinquishes control, control goes back to the DCP telephone.

If an application requests shared control, the telephone stays active.
DCP extension administered without hardware (AWOH) Any DCP type that can be administered for softphone access "X" Not applicable
IP telephone that is logged in Any IP type that can be administered for softphone access Internal software port address, such as S00031, is automatically assigned. If an application requests exclusive control, the IP telephone is logged off. When the application relinquishes control, the IP telephone must be powered off and on again to be logged back on.

If an application requests shared control, the telephone stays active.

IP extension without hardware (i.e., no IP telephone logged in using that extension) Any IP type that can be administered for softphone access Internal software port address, such as S00031, is automatically assigned. Not applicable
